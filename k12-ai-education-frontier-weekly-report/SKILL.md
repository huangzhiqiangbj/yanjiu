# 教育科技与AI前沿周报

## 核心定位
生成一份面向中国教育官员、学校校长和教育研究者的中小学教育科技与AI前沿周报。

这份周报不是业务销售简报，也不是新闻摘抄。重点回答五类问题：
- 本周最值得关注的中小学AI教育信号是什么
- 国际主要国家、国际组织和权威媒体在政策、治理、实施与效果上出现了哪些新变化
- AI新技术、重点报告以及AI技术领域权威专家或CEO的关键观点可能会怎样影响教育、人才培养与就业
- AI教育技术应用领域出现了哪些行业性落地进展，哪些产品、平台或场景已进入可观察的应用阶段
- 国内哪些地区正在形成值得跟踪的AI教育推进做法和阶段性进展

## 默认交付节奏
- 除非用户明确要求 `直接成稿`、`直接输出完整周报` 或 `不要候选直接写`，默认不要一上来写正式周报。
- 默认先输出 `板块化候选条目清单`，由用户决定选哪些资讯，再进入正式成稿。
- 候选条目必须按 `全球AI教育动态`、`AI教育技术应用资讯`、`AI技术前沿及资讯`、`国内“AI+教育”探索` 四个板块分组整理，不要只给混合总表。
- 候选阶段每条默认只写 `标题`、`日期`、`地区/机构`、`一句话理由或事实摘要`，不提前展开正式版 `事件`、`思考与启发` 正文。
- 在用户未明确选定条目前，不生成正式 Markdown、Word 或 HTML 成稿。

## 读者与写作口径
- 面向教育官员：优先解释政策趋势、治理边界、区域推进节奏、规模化条件和评价机制。
- 面向学校校长与管理者：优先解释课程落地、教师培训、校园治理、实施成本、组织方式和限制条件。
- 面向教育研究者：优先解释证据强弱、研究方法、效果数据、国际比较和仍待验证的问题。
- 若用户未指定侧重，默认三类读者兼顾，但保持“政策与治理”优先于“产品热度”。

写作时遵守以下规则：
- 先结论，后事实，再给启发。
- 区分 `事实`、`判断`、`建议`，不要混写。
- 优先使用政策、制度、实践和效果语言，少用营销语言。
- 摘要、标题和结论句默认使用克制判断，不把单周观察写成确定性改革走向；优先使用 `值得关注`、`显示出`、`提示了`、`可考虑`等表达。
- 严禁使用空转起手式或绕弯句式，例如 `不是……，而是……`、`这意味着`、`X的信号是`、`公司公告显示`；只有在确需标明信源等级时，才可使用 `媒体报道`、`公开转引`、`单方口径`。
- 标题和结论句只保留实质信息，不要把“报道方式”或“写作动作”塞进标题；例如能直接写观点时，就不要先写 `公司公告显示`、`媒体称`、`这意味着`。
- 已知有通行中文译名的国际人物、机构、产品，正文首次出现时优先使用中文名；若机构不够知名，首次出现必须补足机构属性，如 `美国非营利机构`、`研究机构`、`公司`、`国际组织`。
- 涉及国外新闻时，标题、候选清单、表格和正文首次出现时，地点和行政区划默认都要写全 `国家 + 地区/州/城市 + 机构属性`，不要只写裸州名、裸城市名或裸机构名。例如优先写 `美国爱荷华州`、`美国哥伦布市学区`、`美国芝加哥市 Alpha School`、`日本文部科学省`、`英国英格兰学校系统`，而不是只写 `爱荷华州`、`哥伦布`、`芝加哥`、`文部科学省`。
- `思考与启发` 不必机械反复写 `对国内中小学`、`国内基础教育而言`；只有在确需落回中国语境时再写，且优先压缩为一句实施提示，不要整段反复锚定。
- 对教育关联度较弱的AI热点，即使来自权威媒体或热门公司，也默认不写入正式周报；只有能清楚回答其与学校治理、课程、教师、评价或人才培养的关系时，才纳入候选。
- 对缺少公开量化数据的事件，不需要专门在 `事件`、`思考与启发` 等相关正文中写 `暂无公开数据`，不要补写想象数字，也不要为了交代数据不足而硬补一个 `成效` 标签行。
- `摘要` 只保留 `本期核心判断`，不再单列 `关键数据速览`。
- 正式交付时，五个主章节必须使用 `一、二、三、四、五` 的大序号；章节内可按需要使用`1、2、3、` 作为次级序号，但不要使用 `1.1`、`3.2` 这类机械层级编号。
- 表格用于快速扫读，不用于承载正文展开。表格单元格默认写成短语或短句，尽量让读者一眼扫完，不要把后文整句解释搬进表格。
- 图表只在能够带来真实信息增量时使用。优先基于资讯正文中的关键数字或跨资讯的非显然数字关系作图；若只是 `入选条目数`、`类别数量`、`国家数量`、`累计发布日期节奏` 这类读者一眼可见的结构统计，默认不要作图。
- 只要正文中出现 `学校数`、`教师数`、`学生数`、`预算`、`课时`、`占比`、`样本量`、`通过率`、`调查比例`、`平台用户规模`、`部署数量` 等关键数据，且图表能帮助读者更快判断差异、结构或趋势，默认必须生成 `有信息量、能看清` 的图表，而不是只停留在表格或文字。
- 图表可读性是硬约束：中文标签、数字、百分比、图题和图内关键值都必须在网页与长图导出中清晰可读；若生成的真实图表字号过小、信息密度过高或单位混乱，必须放大重做，或改成更合适的 HTML 原生信息图/指标卡，不能直接交付。
- 表格单元格默认控制在 `20字以内`；如果信息较多，优先拆成多行或留到正文展开，不要把背景解释硬塞进单元格。
- 默认采用 `简讯体`：除表格和参考来源外，每一条正式资讯的正文整体默认控制在 `200字以内`，且所有条目的 `思考与启发` 都必须写深、写实用；若资讯足够重要，可放宽到 `350字以内`，但必须保持信息密度，不要为了拉长而拉长。
- 各个重点事件中的 `事件` 或 `内容提炼` 默认写成 `1-2 句完整句子`：第 1 句交代发生了什么，第 2 句补充范围、机制、背景或关键动作。
- `思考与启发` 默认写成 `2 句完整句子`：若用户已给出关注点，或事实天然形成清晰主轴，第 `1` 句先把最值得关注的 `3-4` 个关注点压成一句主判断；第 `2` 句再说明制度设计、学校实施、教师能力或研究判断等层面的启发。
- `思考与启发` 的首句主判断尽量控制在 `55字以内`。
- 若某条资讯属于本期核心条目，或单条分析段超过 `180字`，默认拆成 `2个自然段`：第 `1` 段写 `主判断 + 机制解释`，第 `2` 段写 `条件/边界/中国路径`；不要把关键分析挤成单段长文。
- `思考与启发` 保持书面研究语体，避免 `能不能`、`会不会`、`用上`、`更好结果` 这类口语化表达。
- `思考与启发` 不是 `事件` 的改写版；机构数量、覆盖范围、试点数据等若已在 `事件` 中交代过，不要在分析段再复述一遍，除非这些事实本身就是判断核心。
- `思考与启发`优先写清 `谁在推动什么变化`、`影响哪个业务环节`、`为什么重要`，避免既重复事实、又缺少新判断。
- 少用 `深度耦合` 等过于学术化的悬浮术语，优先换成更直接的业务表达。
- 把 `不是……，而是……` 视为一整类禁用句式；`不只是……，而是……`、`不仅……，而是……`、`重点不在……，而在……`、`关键不在……，而在……` 等变体也默认禁用，直接说判断，不要用对举句式制造“深度感”。
- 需要落回中国语境时，尽量直接进入判断，不机械使用 `对中国而言` 这类转场。
- 涉及企业时，说明其作用与边界即可，不强行替企业下行动建议。
- 涉及治理层级、区域推进、课程部署、行政协同或扩散机制时，优先使用 `区域教育治理`、`区域课程工程`、`教育行政体系`、`市—区—校机制` 等教育机制词；除非原始事实本身讨论的是城市战略，不要泛写 `城市系统`、`城市级`、`城市课程`。
- 正文段落默认段首空 `2个中文字符`。Markdown中为避免触发代码块，默认使用 `2个全角空格` 表示，即 `　　**事件：**[正文]`，不要使用半角空格直接缩进，也不要写成 `**事件：**　　[正文]`。注意冒号后不要加空格。
- 避免使用 `重点动态解读`、`重点应用解读`、`重点案例解读` 这类泛化小标题；表格后默认直接进入编号正文。若确需设置小标题，必须使用比“重点解读”更有信息量的名称。
- 默认不单独设置 `数据/证据` 字段；若确需体现可核验数据，应把最关键的 `1-3 个` 数据合并写入 `事件`、`成效`、`对教育的影响` 或表格对应单元格中。
- 合并写入正文或表格的数据，优先保留与教育或AI直接相关的核心数据，如学校数、教师数、学生数、课时、覆盖率、预算、效果指标、训练规模或治理范围。不要为了“看起来有数据”硬写不重要的流程性数字；若公开材料只有这类数字，也要说明其局限。
- `AI教育技术应用资讯` 与国内案例正文中，不同案例默认使用 `1、2、3、` 的顺序编号分开，而不是连续堆叠匿名项目符号。各类正文要点默认不要使用项目符号。尤其是 `事件`、`思考与启发`、`对教育的影响`、`成效`、`内容提炼`、`对人才培养与就业的意义` 这些标签行前，不要加 `-`、`•` 等符号
- Markdown 和 Word 中都不要使用斜体；需要强调时，优先使用加粗。
- `事件`、`思考与启发`、`对教育的影响`、`成效`、`内容提炼`、`对人才培养与就业的意义` 这些行内标签，在 Markdown 和 Word 中默认都要加粗。
- `成效` 字段默认按需出现；只有存在可靠的量化结果、阶段性产出或公开反馈时才单列。若判断为 `暂无公开量化学习效果数据`，不要单独写一行 `成效：暂无公开量化学习效果数据`，直接省略该标签行。
- 不单独设置 `风险边界` 字段；若确需说明限制、前提或待跟踪问题，合并写入 `思考与启发`、`成效` 或 `限制/待跟踪问题`。
- 若某一部分抓到的有效资讯较多，可按日期顺序持续写入，不受示例条数限制。

## 默认范围
- 默认时间窗口为最近 7 天；若用户说“本周”“上周”“本月”，按用户口径执行。
- 对任何时间敏感任务，必须先明确起止日期，再搜索、再核验；超出该时间窗的条目不得写成本期新增资讯。
- 默认聚焦中小学；高教、职教、就业类内容只有在与中小学AI教育、人才培养或升学就业衔接高度相关时才纳入。
- 默认中文输出。
- 默认以中国为主轴，国际视野优先覆盖美国、欧洲、日本，同时积极关注其他国家（如新加坡、韩国、澳大利亚等）的高价值信号，以及具有全球影响力的国际组织和权威研究机构。
- 国际部分默认补扫国外权威媒体对中国AI进展的报道，尤其关注介绍中国AI在教育领域应用、治理、产品落地或学校实践的内容。
- 这里的“欧洲”默认包括欧盟层面与英国等具有高参考价值的欧洲教育治理信号。

## 必须覆盖的五大板块
正式交付默认使用以下五个主章节：

1. `一、摘要`
2. `二、全球AI教育动态`
3. `三、AI技术前沿及资讯`
4. `四、AI教育技术应用资讯`
5. `五、国内“AI+教育”探索`

每个板块都必须争取三种表达：
- `数据`：人数、学校数、地区数、课时、预算、覆盖面、效果指标、调查结果等可核验信息
- `表格`：用于横向比较和快速阅读，默认保持简洁，不重复正文
- `文字`：用于解释背景、分析趋势和提出启发

## 信号优先级
优先保留以下内容：
- 国际政策与治理：国家教育政策、课程标准、AI使用规则、数据与伦理要求、教师培训制度、评估框架
- 国际实施与效果：学区、学校系统、国家项目、评价结果、覆盖规模、教师与学生反馈
- 国外权威媒体观察：`纽约时报`、`华尔街日报`、`金融时报`、`彭博`、`路透`、`经济学人`、`Nature`、`Science` 等对中国AI进展的报道，尤其是介绍中国AI在教育领域应用的内容
- AI技术进展：本周出现且可能影响教学、评价、教研、治理和学习支持的新技术或重要能力变化
- 重点报告与人物：国际组织、研究机构，以及AI技术领域权威专家、首席科学家、研究负责人、企业CEO或官方负责人关于AI教育、人才培养和就业的关键判断
- AI教育技术应用：AI教学平台、教师工作流工具、智能测评、学习支持系统、校园管理系统、区域级部署与采购等行业性落地进展
- 中国地方探索：中央教育部、省市级教育行政部门、区域项目、跨校机制、课程体系、教师发展、校园治理和阶段性成效
- 本期热门动态：本周在权威媒体、行业媒体、教育科技平台或头部机构公告中被集中讨论，且与课程、教师、评价、治理、平台或人才培养直接相关的高讨论度事件

低优先级内容：
- 与K-12关系弱的泛AI热点
- 缺少教育场景证据的单方宣传
- 只有概念没有实施动作或效果信息的报道
- 单个学校、单个区的零散宣传或样板活动，除非具全国示范性或用户明确要求学校案例
- 只有“热度”没有明确教育关联点的科技新闻

## 信源优先级与核验规则
信源优先级：
1. 官方与监管信源
2. 学校、学区、国际组织、研究机构
3. 权威媒体与专业媒体
4. 公司公告、产品博客、单方口径

核验规则：
- 对“本周”“过去 7 天”“某日期以来”等时间敏感任务，搜索结果摘要只能用来建候选池，不能直接入稿。
- 每条正式入选内容至少核验三项：`发布日期`、`发布主体`、`正文是否支持标题结论`。
- 若使用非一级官方来源，要标注 `媒体报道`、`公开转引` 或 `单方口径`。
- 任何超出时间窗的条目，即使本周被转载或再次传播，也不得写成本期新增资讯；若确需引用，只能作为背景补充，且要明示其原始日期。
- 对效果类表述特别谨慎；如果只有“提升显著”“效果很好”等定性措辞，没有方法和指标，不要写成确定性结论。

## 国际覆盖要求
- 国际部分优先覆盖 `美国`、`欧洲`、`日本` 三个方向，但不要局限于此。若其他国家（如新加坡、韩国等）有极具启发性的高价值信号，也应积极纳入。
- 除国家信号外，还要补充 `国际组织`、`权威研究机构` 和 `权威媒体` 的高价值信息。
- 国际部分重点关注四类主题：`政策治理`、`学校实施`、`效果证据`、`人才培养与就业影响`。

## 国外权威媒体补扫规则
- 默认补扫 `纽约时报`、`华尔街日报`、`金融时报`、`彭博`、`路透`、`经济学人`、`Nature`、`Science` 等国外权威媒体与高影响力科学媒体。
- `Nature`、`Science` 及其子刊、新闻页、专题页可进入候选池；若条目属于 `partner content`、`sponsored content`、`branded content` 或其他付费合作内容，必须显式标注广告属性，并按 `证据强度` 与 `信源等级` 降级处理，不能和官方政策、学校系统公告或独立新闻报道同级引用。
- 重点关注其对中国AI进展的报道，尤其是介绍中国AI在教育领域应用、治理、产品落地或学校实践的内容。
- 若该类资讯与本期时间窗一致、且能核验发布日期与正文细节，应写入第二部分 `全球AI教育动态`，并明确标注为 `国外媒体视角` 或 `媒体报道`。

## 中国区补扫规则
- 只要中国在覆盖范围内，除了教育部和全国性媒体，还必须补扫地方教育局、学校和区域项目。
- 默认优先覆盖：`北京`、`上海`、`深圳`、`广州`、`重庆`、`杭州`、`成都`、`南京`。
- 不要只用 `AI`、`中小学`、`学校` 等关键词，还要补扫 `教育强国`、`教育数字化`、`基础教育`、`综合改革试点`、`教师队伍建设`、`国家智慧教育平台` 等顶层表述。
- 若地方官网抓取失败，不直接放弃；先保留候选，再用 `政府转载`、`官微`、`党媒原创` 或 `权威媒体` 做二次核验。

## 每条入选内容都要回答的五问
1. 发生了什么
2. 有哪些可核验数据或证据
3. 为什么与中小学AI教育有关
4. 对制度设计、学校实施、教师能力或研究判断有什么启发
5. 还存在哪些限制或待跟踪问题

## 工作流程
### 1. 确认任务边界
- 明确时间窗、地区、学段、读者类型和重点主题。
- 若用户未说明，使用默认范围。

### 1.1 默认先给候选，不急于成稿
- 对常规周报请求，默认先进入 `候选条目阶段`，不要在未确认条目前直接写正式周报。
- 候选条目阶段必须按 `全球AI教育动态`、`AI教育技术应用资讯`、`AI技术前沿及资讯`、`国内“AI+教育”探索` 四个板块整理。
- 若某个板块在目标时间窗内没有合格条目，必须明确说明是 `时间窗不符`、`证据不足` 还是 `教育关联度不足`，不能静默缺席。
- 只有当用户明确回复要写哪些条目，或明确授权“按你推荐的写”时，才进入正式成稿阶段。

### 2. 建立候选池
- 先尽可能广地建立候选池，为后续筛选保留足够空间；正式入选内容不设硬性数量上限。
- 候选池至少记录：`来源`、`日期`、`地区/机构`、`类别`、`中小学关联点`、`数据点`、`信源等级`、`是否入选`、`未入选原因`。
- 候选池最好补记 `层级`，如 `国家级`、`区域级`、`学校级`；正式成稿默认优先使用国家级和区域级信号。
- 候选池默认拆成两层：`系统级信号池` 与 `热门动态池`。前者负责政策、治理、区域推进和平台基础设施，后者负责本周讨论度高、但仍与教育直接相关的产品发布、合作、扩围、重要人物表态、重点报告和行业落地进展。
- 国际候选优先覆盖美国、欧洲、日本和国际机构，并兼顾其他国家，也要补入国外权威媒体关于中国AI教育应用的高价值报道；国内候选要覆盖中央与地方。
- `热门动态池` 不能因为系统级条目已经足够就省略；每期至少补扫一轮，并记录每条的 `热度来源`，如 `权威媒体集中报道`、`行业媒体重复出现`、`头部机构发布`、`教育圈高讨论度`。

### 2.1 热门动态补扫规则
- 热门动态默认补扫四类来源：`国外权威媒体`、`教育专业媒体`、`头部机构/公司公告或博客`、`区域或行业级扩围与合作新闻`
- 热门动态的重点对象包括：大厂教育产品、教师工作流工具、学生学习支持平台、重要合作或试点扩围、关键融资与并购、教育相关人工智能训练计划、重大报告与人物表态
- 一条热门动态要进入正式候选，至少满足以下两项：
　　1. 与课程、教师、评价、治理、平台或人才培养有直接关系
　　2. 来自权威媒体、教育专业媒体、官方机构或头部机构公告
　　3. 正文能核验出明确机制、规模、部署动作或训练安排
　　4. 能稳定产出有用的 `思考与启发`
- 若某条热门动态最终不入稿，也要在候选池中留下未入选原因，避免因为“感觉不重要”而被静默漏掉

### 3. 做时间与事实核验
- 优先看原始页面，再看转载页面。
- 不能只看标题；要确认正文是否真的包含标题声称的政策、效果或数据。
- 对时间敏感任务，必须逐条核对原始发布日期，超出起止日期的条目不得作为本期新增资讯写入。
- 对带有“调查显示”“研究发现”“试点成效”的条目，优先补看方法、样本或适用范围。

### 4. 先按四个板块给用户挑选
- 在正式写周报前，先把候选条目按 `全球AI教育动态`、`AI教育技术应用资讯`、`AI技术前沿及资讯`、`国内“AI+教育”探索` 四个板块聚类展示给用户。
- 每个板块默认给出 `3-8` 条候选；若有效条目不足，可少于该数量，但要写明原因。
- 每条候选至少包含：`候选序号`、`标题`、`日期`、`地区/机构`、`推荐度或总分`、`一句话入围理由`。
- 候选阶段只做事实型简介，不提前展开正式版 `事件`、`思考与启发`、`对教育的影响` 或 `对人才培养与就业的意义`。

### 4.1 用户确认后再进入正式周报
- 用户选定条目后，先复述并锁定名单，再进入正式成稿。
- 若用户只选了部分板块，可按其选择成稿；若用户未限制数量，默认优先按其选择中最重要的 `5-10` 条组织正式周报。
- 若用户回复“按你推荐的来”，才能由 AI 自行从候选池中锁定正式名单。

### 5. 先搭表格，再写文字
- 这一阶段默认发生在 `用户已确认条目` 之后，而不是候选阶段之前。
- 先完成国际动态总览表、AI教育技术应用总览表、国内案例总览表。
- 若第三部分内容足够，再补技术/报告观察表。
- 表格完成后再写摘要判断和分节解读，避免文字与数据脱节。
- 表格只保留“读者扫一眼就能抓住”的摘要信息；更完整的背景、机制和启发放到正文里展开。
- 表格单元格默认保持短句，尽量不超过 `20字`。
- 若正文中出现覆盖学校数、教师数、学生数、预算、课时、用户规模、参与机构数、比例变化或调查结果等关键数字，且图表明显优于文字或表格表达，必须主动读取 sibling skill [visualization](../visualization/SKILL.md) 并生成真实图表；若没有这类数字，默认不为凑版面硬作图。
- 若真实图表生成后出现 `中文标签难辨认`、`数字太小`、`同图混放不同量纲导致误读`、`小比例数据被挤压看不清` 等情况，必须二次调整；必要时可放弃通用图表，改用更适合阅读的 `HTML 原生大字号图表 / 指标面板 / 进度条信息图`。

### 5.1 分析段标准调用流程
- 在撰写 `思考与启发`、`对教育的影响`、`对人才培养与就业的意义` 前，先读取 sibling skill [education-insight-architect](../education-insight-architect/SKILL.md)。
- 先为每条待写分析的资讯整理事实包，至少包含：`标题`、`1-2句事实摘要`、`3-6条已核验事实`、`板块类型`、`信源等级`、`是否本期核心条目`。
- 在进入正式写作前，先额外提炼这条资讯最值得关注的 `3-4` 个主轴；若用户已明确给出关注点，优先直接吸收，不要改写成更泛的抽象判断。
- 若首句主判断超过 `55字`，优先先删减，再进入正式写作。
- 若条目来自国外，正式写作前还要补问一句：它对中国基础教育推进路径的启发是什么，哪些做法可借鉴、哪些不能照搬。
- 若条目涉及企业参与教育变革，还要补问一句：企业在这条资讯中承担的是能力支持、平台供给、教师培训还是实施服务角色，其价值与边界分别是什么。
- 若同一版周报中需要写分析段的条目不超过 `3` 条，且主任务难点仍在事实组织，可不启用 subagent，但仍必须对每一条都按 `education-insight-architect` 的句式约束和自检清单深写分析段。
- 满足任一条件时，优先启用只读 subagent 批量生成分析段：
　　1. 同一版周报有 `4` 条及以上条目需要统一写分析段
　　2. 用户明确要求“写深”“写透”“更像架构师”
　　3. 本期核心条目需要放宽到 `350字以内`
　　4. 主任务瓶颈已从事实核验转为分析提炼
- 启用 subagent 时，优先使用 `generalPurpose`、`readonly: true`，并直接复用 `education-insight-architect` 中的 `Subagent 提示词模板`。
- subagent 只产出分析字段，不得补新事实、改标题、扩写背景；主 skill 在回收结果后再统一做长度、语气、命名和禁用句式检查。

### 5.2 分析段字面层质检
- 在任何分析段进入正式成稿前，必须再做一轮 `literal-level lint`，不要只凭“已经读起来差不多”就结束。
- 长度检查：每条资讯正文整体默认控制在 `200字以内`，核心条目可放宽到 `350字以内`；但字数上限是 `ceiling`，不是默认目标。若草稿虽然没超字数，但 `思考与启发` 仍像摘要体、还没写出机制、条件或边界，就继续补足。
- 分段检查：若条目属于本期核心事件，或单条分析段超过 `180字`，默认拆成 `2个自然段`；首段必须同时交代 `主判断 + 机制解释`，次段再写 `条件/边界/中国路径`，不要把关键分析压成单段长文。
- 禁用句式搜索：交付前逐项检查并删除 `不是……，而是……`、`不只是……，而是……`、`不仅……，而是……`、`重点不在……，而在……`、`关键不在……，而在……`、`这意味着`、`X的信号是`、`公司公告显示` 等句式及其变体。
- 教育术语精度检查：若一段话本质在谈治理层级、区域推进、课程部署、采购约束或扩散机制，优先回到教育机制词，如 `区域教育治理`、`区域课程工程`、`教育行政体系`、`市—区—校机制`；只有当地理尺度本身就是判断核心时，才保留 `城市级` 等说法。

### 6. 深度启发与研判生成规则（必须严格执行）
这是本 skill 的核心约束之一，优先级高于一般修辞偏好和常规压缩习惯。凡是输出 `思考与启发`、`对教育的影响`、`对人才培养与就业的意义`等，都必须实质执行本节规则，不能把启发写成事实复述、态度表态或空泛总结。
- 在正式撰写这些分析段前，优先读取 sibling skill [education-insight-architect](../education-insight-architect/SKILL.md)，并按其中的输入要求、句式约束、自检清单和 subagent 提示词模板执行。
- 杜绝空泛修饰：严禁使用值得大力推广、意义十分深远、亟需引起重视等无实质信息的套话。
- 强制采用 `架构师视角`：优先解释这条资讯反映了什么制度设计、能力迁移、业务流重构或治理逻辑变化，不要把启发写成评论员式表态。
- 默认采用 `主判断 + 机制/条件` 的两步式启发：第 `1` 句优先把这条资讯最值得关注的 `3-4` 个主轴压成一句总判断，回答“这条资讯最该关注什么”；第 `2` 句回答“它通过什么制度、协同或资源配置机制发挥作用，以及落地需要补哪些平台、规则、师训、权限或评价条件”。
- 对国外热点，不能停留在海外案例本身；优先补出其对中国教育发展路径的启发、制度差异和本土化边界，看到 `政府主导、区域推进、平台支撑、教师培训前置、资源均衡优先` 的中国路径。
- 对企业参与，不能只写商业动作或风险提醒；要说明企业如何在公共教育体系中承担技术、产品、培训、服务和实施支持作用，并同时指出其不能替代政府治理、学校主体和教育目标设定。
- 强制调用专业分析框架：在生成思考与启发时，AI必须根据资讯的具体属性，从以下七个维度中灵活选取1至2个最契合的视角进行深度交叉拆解：
　　1. 认知与学习科学：是否触及高阶思维与探究能力培养。
　　2. 素养导向与评价协同：能否对接新课标，是否支持过程性与增值性评价。
　　3. 人机协同与教师演进：如何帮助教师剥离低效劳动并向情感与复杂认知引导转型。
　　4. 核心业务流重塑：是否真正契合备课、授课、批改等真实场景，避免新增数字负担。
　　5. 规模化常态部署条件：从试点走向全面铺开所需的网络、硬件与师资培训最低门槛。
　　6. 成本结构与资源分配：包含算力与授权等显性财务成本，以及课程适配等隐性时间投入。
　　7. 技术普惠与教育均衡：考察其对弥合校际差距的潜力，或防范数字鸿沟扩大的风险。
- 若初稿中的启发只是在重复事实、泛泛提醒风险，或没有体现上述分析框架，就继续推敲，不要直接交付。
- 提供建设性的实施提示：在分析落地限制时，焦点应放在防范沉没成本和提示实施前提上。客观指出常态化应用需要哪些配套支撑，或建议在哪些环节开展小规模验证，为管理者提供安全可行的过渡路径。
- 务实克制的趋势推演：对于核心标志性事件，允许适度展开逻辑推演。避免使用颠覆、彻底重构等夸大词汇，重点预测该趋势在未来1至2年内。
- 适度放宽限制：所有条目的分析都必须写深写透；对于本期最核心的1至2个大事件，仅放宽其整条资讯长度到 `350字以内`。放宽后的新增篇幅应优先用于展开结构性分析、实施前提或治理边界，而不是重复背景信息。

### 7. 默认落盘
- 除非用户明确要求“只在对话里展示、不落盘”，否则正式交付默认先在 `docs/周报/` 下为本次生成创建独立文件夹。
- 文件夹名优先使用 `K-12教育科技与AI前沿周报_YYYYMMDD_YYYYMMDD`。
- 若 `docs/周报/` 中已存在同名文件夹，不要覆盖原文件夹；必须按同目录递增后缀创建 `K-12教育科技与AI前沿周报_YYYYMMDD_YYYYMMDD_2`、`..._3` 等，选择首个未被占用的文件夹名。
- Markdown 默认保存为 `docs/周报/[批次文件夹名]/[批次文件夹名].md`。
- 生成 Word 时，必须保存为 `docs/周报/[批次文件夹名]/[批次文件夹名].docx`，与 Markdown 保持同一文件基名。
- 如用户额外要求领导速览版、微信短版、HTML 页面或长图版，也默认放入同一批次文件夹，不要散落到 `docs/周报/` 根目录；若需另存短版，也应在同一批次文件夹内完成。

### 8. 要主动输出 Word
- 在保存 Markdown 的同时，导出同一批次文件夹内的同名 `.docx` 文件；若批次文件夹名因重名被另存为 `_2`、`_3` 等，Word 也必须使用相同后缀。
- Word 默认样式：文档标题使用 `黑体`、`加粗`、`二号`、`居中`；标题下方日期另起一行，使用月日格式，不写年份， `黑体`、`二号`、`居中`。
- Word 中 `一、二、三、四、五` 五个主章节标题使用 `黑体`、`加粗`、`三号`；`###` 次级标题使用 `仿宋`、`加粗`、`三号`；正文默认使用 `仿宋`、`三号`，除非用户明确要求，否则不要导出项目符号。
- Markdown 正文段落默认使用 `2个全角空格` 起段；导出 Word 时，正文与 `###` 次级标题段落默认均设为首行缩进 `2字符`。若使用 `事件`、`思考与启发`、`成效`、`内容提炼`、`对教育的影响` 等标签，缩进应出现在标签前，写成整段段首空两格，不要把空两格放在标签后。
- Word 正文与 `###` 次级标题默认使用 `两端对齐`。
- Word 全文默认使用固定 `28磅` 行距。
- 表格表头和表格正文必须使用 `仿宋`、`四号`，表格内部必须使用 `1.0` 单倍行距，不沿用正文的 `28磅` 固定行距。
- `附：参考来源` 默认可保留在 Markdown 中，但不强制写入 Word；如无特别要求，Word 可省略该部分。
- Word 文档中不要出现斜体；若 Markdown 中存在会触发斜体的语法，导出前应改写为普通文本或加粗文本。
- 若表格导入 Word 后显得过宽，优先缩短表格文字，不要依赖缩小字号来塞入过长内容。
- 导出 Word 时应尽量避免表格跨页；若单张表过长，优先拆表或压缩措辞，避免同一行被分页切断。

## 输出骨架

- 默认已直接在 `docs/周报/` 下创建独立批次文件夹，并将 Markdown 和 Word 落入同一文件夹；若存在同名批次，已自动改用 `_2`、`_3` 等递增文件夹名
- Word 标题使用 `黑体`、`加粗`、`二号`、`居中`，日期另起一行且只写月日， `黑体`、`二号`、`居中`；`一、二、三、四、五`标题使用 `黑体`、`加粗`、`三号`
- Word 全文使用固定 `28磅` 行距；正文使用 `仿宋`、`三号`，且不要误导出项目符号；正文与 `###` 次级标题默认两端对齐
- Word 表格已使用 `仿宋`、`四号`、`1.0` 行距，并尽量避免跨页
- `事件`、`思考与启发`、`对教育的影响`、`成效` 等标签已加粗；`附：参考来源` 默认可不写入 Word
- 不要用 `不是……，而是……` 及其变体，例如 `不只是……，而是……`、`重点不在……，而在……`；直接说观点。
