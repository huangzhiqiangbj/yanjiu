# 全球AI教育动态周报

## 核心定位
- 先广泛建池，再逐条评分，先把候选资讯名单告诉用户，不要急着生成成稿。
- 只有在用户明确选定要写的 `5` 条资讯后，才进入正式撰写、Markdown/Word/HTML 导出。
- 在候选选择阶段，默认先按 `全球AI教育动态`、`AI教育技术应用资讯`、`AI技术前沿及资讯`、`国内“AI+教育”探索` 四个板块展示候选池，再给出汇总推荐清单，不要只给一张混合排序表。
- 评分用于后台筛选、排序和取舍；在 `候选选择阶段` 可以展示总分或推荐等级，方便用户挑选，正式成稿默认不显示评分。
- 正式交付不再按 `全球AI教育动态`、`AI技术前沿及资讯`、`AI教育技术应用资讯`、`国内“AI+教育”探索` 分章节。
- 用户最终选定的 `5` 条资讯可以来自任何类别，但必须能清楚回答“为什么值得教育系统优先关注”。
- 这不是单纯的热度榜，而是 `教育相关性 + 系统重要性 + 证据强度 + 启发价值 + 时效讨论度` 的综合筛选。
- Markdown、Word、HTML 的对外标题、页面辅助文字和栏目标题统一使用 `全球AI教育动态周报`、`资讯总览` 等中性表述，不再出现“精选榜”“Top5”等外显标题词。
- 只有当资讯正文本身提供了关键数字，或多条资讯之间存在基于原始数字的非显然比较关系时，才考虑作图；若只是 `入选条目数`、`类别数量`、`国家数量`、`累计出现节奏`、`发布日期密度` 这类读者一眼可见的结构统计，默认不要作图。
- 只要候选池或正式 `5` 条中出现 `学校数`、`教师数`、`学生数`、`预算`、`课时`、`调查比例`、`通过率`、`样本量`、`平台用户规模`、`部署数量` 等关键数据，且图表能帮助读者更快看出差异、结构或趋势，默认必须生成 `有信息量、能看清` 的图表，而不是只停留在表格或文字。
- 当候选池或正式周报中存在适合图表表达、且图表能提供真实信息增量的数据时，必须主动读取 sibling skill [visualization](../../visualization/SKILL.md)，再调用其中合适的图表能力生成真实图表，而不是只用文字说“可视化显示”或只靠普通表格替代。
- 图表可读性属于硬约束：中文标签、数字、百分比、图题和图内关键值必须在网页与导出长图里清晰可读；若真实图表字号过小、信息过挤或量纲混乱，必须放大重做，或改用更适合阅读的 HTML 原生信息图、指标卡或进度条图。

## 读者与写作口径
- 面向教育官员：优先解释政策趋势、治理边界、区域推进节奏、规模化条件和评价机制。
- 面向学校校长与管理者：优先解释课程落地、教师培训、校园治理、实施成本、组织方式和限制条件。
- 面向教育研究者：优先解释证据强弱、研究方法、效果数据、国际比较和仍待验证的问题。
- 若用户未指定侧重，默认三类读者兼顾，但保持 `政策与治理` 优先于 `产品热度`。

写作时遵守以下规则：
- 先结论，后事实，再给启发。
- 区分 `事实`、`判断`、`建议`，不要混写。
- 优先使用政策、制度、实践和效果语言，少用营销语言。
- 标题和结论句直接写判断，不要写 `公司公告显示`、`媒体称`、`这意味着`、`X的信号是` 等空转起手式。
- 已知有通行中文译名的人物、机构、产品，正文首次出现时优先使用中文名；若不够知名，首次出现要补足机构属性。
- 涉及国外新闻时，标题、候选清单、表格和正文首次出现时，地点和行政区划默认都要写全 `国家 + 地区/州/城市 + 机构属性`，不要只写裸州名、裸城市名或裸机构名。例如优先写 `美国爱荷华州`、`美国哥伦布市学区`、`美国芝加哥市 Alpha School`、`日本文部科学省`、`英国英格兰学校系统`，而不是只写 `爱荷华州`、`哥伦布`、`芝加哥`、`文部科学省`。
- `思考与启发` 不机械重复 `对国内中小学而言`。只有需要落回中国语境时，再压缩写一句实施提示。
- 对教育关联度偏弱的 AI 热点，即使媒体热度高，也不要因为“热门”而挤入正式 `5` 条。
- `思考与启发` 采用双下限 + 双上限：普通条目严格控制在 `150-200字`，核心 `1-2` 条严格控制在 `250-350字`。低于对应下限，通常说明还停留在摘要体，不能直接交付。
- `事件` 默认写成 `1-2` 句完整句子；`思考与启发` 默认写成 `2` 句完整句子。若用户已给出关注点，或事实天然形成清晰主轴，第 `1` 句先把最值得关注的 `3-4` 个关注点压成一句主判断，第 `2` 句再解释机制、条件或治理含义。
- `思考与启发` 的首句主判断尽量控制在 `55字以内`。
- 核心 `1-2` 条若因篇幅增加而拆成 `2` 个自然段，第 `1` 段必须同时包含 `主判断 + 至少1句机制解释`，不要把主判断单独切成一句成段；第 `2` 段再展开实施重点、边界或后续观察点。
- 若单条关键分析超过 `180字`，即使不是本期最核心条目，也默认拆成 `2` 个自然段，不要把关键信息压成单段长文。
- `思考与启发` 保持书面研究语体，避免 `能不能`、`会不会`、`用上`、`更好结果` 这类口语化表达。
- `思考与启发` 不是 `事件` 的改写版；若覆盖范围、试点数据、机构数量等已在 `事件` 中交代过，不要在分析段再次复述，除非这些事实本身就是判断核心。
- `思考与启发` 要删繁就简，优先写清 `谁在推动什么变化`、`影响哪个业务环节`、`为什么重要`。
- 对中央部委、省级教育主管部门、正式部署会、纲要或行动计划类条目，`思考与启发` 优先写政策抓手、实施重点、协同机制和推进节奏，不把分析写成对政策本身的质疑、审判或居高临下式评判。
- 若中央政策或平台升级反复强调 `全要素 / 全过程 / 全场景`，优先识别并写明 `AI已被界定为教育系统公共能力，而非课堂里的单点工具`；分析时要落到课程供给、教师支持、数据治理、服务触达等公共能力维度。
- 涉及平台升级或公共服务平台改版时，优先判断其价值是否进入 `备课、学习支持、教师发展、治理服务` 等真实业务流，不把“模型接入”本身误写成主要结论。
- 若条目显示治理边界、数据可用范围、关键决策保留权开始进入正式文件或制度安排，要明确点出其代表 `教育AI从试验逻辑转向制度逻辑`，而不是仅写成一般性推进。
- 少用 `深度耦合` 等过于学术化的悬浮术语，优先换成更直接的业务表达。
- 把 `不是……，而是……` 视为一整类禁用句式；`不只是……，而是……`、`不仅……，而是……`、`重点不在……，而在……`、`关键不在……，而在……` 等变体也默认禁用，直接写判断。
- 对国外热点，优先补一句其对中国教育发展路径的启发、可借鉴之处或不宜照搬的边界。
- 需要落回中国语境时，尽量直接进入判断，不机械使用 `对中国而言` 这类转场。
- 对企业参与，优先说明其在公共教育体系中的支持作用，不把企业简单写成商业机构或万能主体。
- 涉及企业时，说明其作用与边界即可，不强行替企业下行动建议。
- 涉及治理层级、课程部署、行政协同或扩散机制时，优先使用 `区域教育治理`、`区域课程工程`、`教育行政体系`、`市—区—校机制` 等教育机制词；除非原始事实本身讨论的是城市战略，不要泛写 `城市系统`、`城市级`、`城市课程`。
- Markdown 正文段落默认段首空 `2个全角空格`，写成 `　　**事件：**[正文]`，不要把空两格放在标签后。
- Markdown 和 Word 中都不要使用斜体；需要强调时优先使用加粗。

## 默认范围
- 默认时间窗口为最近 `7天`；若用户说“本周”“上周”“本月”，按用户口径执行。
- 对任何时间敏感任务，必须先明确起止日期，再搜索、再核验；超出时间窗的条目不得写成本期新增资讯。
- 默认聚焦中小学；高教、职教、就业类内容只有在与中小学AI教育、人才培养或升学就业衔接高度相关时才纳入。
- 默认中文输出。
- 默认以中国为主轴，国际视野优先覆盖美国、欧洲、日本，同时积极关注其他国家、国际组织和权威研究机构。
- 虽然正式成稿不再分模块，但建池阶段仍要覆盖 `国际政策治理`、`AI技术与报告`、`AI教育技术应用`、`国内地方探索` 四类信号，避免最终 `5` 条失真。
- 默认必须显式补扫以下 `6` 条信源线，不得只搜到中美就停：`教育部及直属权威源`、`中国权威媒体`、`欧洲官方教育治理源`、`日本官方教育源`、`国际组织`、`国外权威媒体与高影响力科学媒体对中国AI教育/中国AI+教育的报道`。
- 若某一条信源线在目标时间窗内没有合格命中，也必须在候选阶段明确说明“已搜索但本期未找到满足时间窗与核验要求的条目”，不得静默缺席。
- 搜索语言必须随国家/地区切换，不得默认全部用英文或全部用中文：
  - `中国`：默认使用 `中文` 搜索，关键词优先写 `人工智能`、`教育`、`中小学`、`教育部`、`国家智慧教育平台`。
  - `日本`：默认使用 `日语` 搜索，关键词优先写 `生成AI`、`学校`、`教育`、`文部科学省`、`教科書`、`リーディングDXスクール`。
  - `欧洲/欧盟/英国/美国/国际组织/国外权威媒体`：默认使用 `英语` 搜索，关键词优先写 `AI education`、`schools`、`teachers`、`students`、`policy`、`governance`。
  - 若某个国家或地区存在通行本地语且英语结果明显稀缺，可追加一轮本地语补扫，但必须先完成该地区的默认语言搜索。

## 候选池要求
- 正式评分前，先建立足够宽的候选池；若时间窗内有效资讯充足，候选池默认争取达到 `12-20` 条。
- 候选池至少拆成两层：`系统级信号池` 与 `热门动态池`。前者负责政策、治理、区域推进和平台基础设施，后者负责本周讨论度高、但仍与教育直接相关的产品发布、合作、扩围、人物表态和行业落地进展。
- 每条候选至少记录以下字段：`来源`、`日期`、`地区/机构`、`类别`、`1-2句事实摘要`、`中小学关联点`、`关键数据`、`信源等级`、`五维得分`、`总分`、`是否入选`、`未入选原因`。
- 若候选池不足 `10` 条，不要急于交付；继续补扫，除非目标时间窗内确实没有更多高价值条目，并在交付时如实说明。
- 搜索结果摘要只能用于建池，不能直接进入评分和正式成稿。
- 排序后用于呈现给用户的候选清单，默认优先展示 `8-12` 条；若有效高质量候选本来就很少，可少于 `8` 条，但要说明原因。
- 建池时，`教育部/国家智慧教育平台/中国教育报/新华社/人民日报` 这一组国内高权威信号必须单独检查一次；即使最终不入选正式 `5` 条，也要先进入候选池备查，除非明确不在时间窗内或与教育关联过弱。
- 对 `教育部` 线，凡是出现 `部署会`、`推进会`、`战略行动`、`国家智慧教育平台`、`人工智能+教育`、`教育数字化`、部长讲话、平台新版本上线、中心发布等关键词的条目，默认按高优先级候选处理，优先级高于一般标准发布、术语发布或评论性转载。
- 建池时，`欧洲`、`日本`、`国际组织` 三条线至少各完成一次原始信源搜索或核验；如果没有进入候选清单，必须写明是因为 `时间窗不符`、`证据不足` 还是 `教育关联度不足`。
- 建池时，必须额外补扫 `国外权威媒体对中国AI教育/中国AI+教育的报道`，优先检查 `Reuters`、`Financial Times`、`New York Times`、`Wall Street Journal`、`Bloomberg`、`The Economist`、`Nature`、`Science` 等；若目标时间窗内无合格条目，也必须明确说明“已补扫但未命中”，不要让用户误以为未搜索。
- `Nature`、`Science` 及其子刊、新闻页、专题页可进入候选池；若条目属于 `partner content`、`sponsored content`、`branded content` 或其他付费合作内容，必须在候选阶段明确标注其广告属性，并按 `证据强度` 和 `信源等级` 降级处理，不能与官方政策、学校系统公告或独立新闻报道同级引用。

## 评分规则
对每一条候选资讯做 `100分制` 打分。默认使用以下权重：

| 维度 | 分值 | 核心判断 |
| --- | --- | --- |
| 教育相关性 | 0-25 | 是否直接影响课程、教师、评价、治理、平台或人才培养 |
| 系统重要性 | 0-25 | 是否体现国家/区域级动作、制度变化、规模化部署或关键能力跃迁 |
| 证据强度 | 0-20 | 是否能核验日期、主体、正文细节，并有公开数据、机制或范围支撑 |
| 启发价值 | 0-20 | 是否能稳定产出对政策、学校实施或研究判断有用的深度启发 |
| 时效与讨论度 | 0-10 | 是否处在目标时间窗内，且具备真实讨论度或节点意义 |

评分时额外遵守：
- `教育相关性` 低于 `15` 分的条目，默认不进入正式 `5` 条，除非用户明确要求关注纯技术动态。
- `证据强度` 低于 `8` 分的条目，默认只留在候选池备查，不进入正式成稿。
- 同一事件的不同转载或不同媒体跟进，不得重复占据多个正式名额。
- 若总分接近，优先保留 `信源等级更高`、`规模更大`、`机制更清楚`、`对中国教育语境更有可借鉴性` 的条目。
- 默认尽量保持正式 `5` 条的信息多样性，但不要为了“平衡类别”牺牲真正更重要的条目。

## 工作流程
### 1. 确认任务边界
- 明确时间窗、地区、学段、读者类型和重点主题。
- 若用户只说“做一份精选周报”，默认沿用本 skill 的时间窗和读者口径。

### 2. 广泛建池
- 先广泛搜索并记录候选，而不是边搜边写定稿。
- 国际部分优先覆盖 `美国`、`欧洲`、`日本`、`国际组织`、`权威研究机构` 和 `国外权威媒体`。
- 应用部分继续覆盖学校、学区、区域平台、教师工具、测评工具、学习支持系统和校园治理场景。
- 国内部分继续覆盖教育部、省市教育局、区县教育局、区域项目和具有示范性的学校系统。
- 建池搜索必须按 `国家-语言矩阵` 执行，不能把一个英文查询复制到所有国家，也不能只用中文搜完全部地区。
- 对中国部分，除一般搜索外，必须专项补扫：`教育部官网`、`国家智慧教育平台`、`中国教育报/中国教育新闻网`、`新华社/人民日报/人民网教育`。
- 教育部官网搜索时，必须额外补扫以下高命中关键词组合：`国家教育数字化战略行动`、`部署会`、`国家智慧教育平台`、`人工智能+教育`、`AI for学校教育`、`AI for教师发展`、`AI for教育治理`，避免只命中边缘性标准或评论稿而漏掉部级部署信号。
- 对欧洲部分，优先补扫：`European Commission / European Education Area / EACEA / GOV.UK / Eurydice`。
- 对日本部分，优先补扫：`文部科学省`、`リーディングDXスクール`、`生成AIパイロット校`、地方教育委员会或学校系统公开页。
- 对国际组织部分，优先补扫：`UNESCO`、`OECD`、`UNICEF`、`World Bank`、相关教育规划或数字教育项目页。
- 对外媒涉华部分，除了搜全球 AI 教育动态，还要专门搜 `中国AI教育`、`China AI education`、`China AI in schools`、`China education AI policy` 等关键词，不得默认国外媒体不会报道中国。
- 候选池构建时，至少保留一份 `已执行搜索语种记录`：如 `中国=中文`、`日本=日语`、`欧盟/美国/国际组织=英语`。若最终交付里出现某一国家或地区缺席，需要能回溯说明当时使用了什么语言搜索、为何未命中。

### 3. 做时间与事实核验
- 至少核验三项：`发布日期`、`发布主体`、`正文是否支持标题结论`。
- 优先看原始页面，再看转载页面；不能只看标题。
- 对效果类表述特别谨慎；如果只有“提升显著”“效果很好”等定性措辞，没有方法和指标，不要给高分。

### 4. 逐条打分并排序
- 对候选池中的每一条都进行五维评分，并写一句简短评分理由。
- 评分记录保留在后台候选池，用于排序、去重和边界取舍，不默认写入正式成稿。
- 排序后先做去重，再形成 `候选推荐清单` 和 `备选清单`。
- 未入选条目要保留 `未入选原因`，避免静默遗漏。

### 4.1 先把候选资讯交给用户选择
- 在这一阶段，默认不要直接写 `本期判断`、`事件`、`思考与启发` 正文，也不要直接开始导出 Markdown、Word、HTML。
- 先向用户展示一个可选择的候选清单，至少包含：`候选序号`、`资讯标题`、`日期`、`地区/机构`、`类别`、`总分或推荐等级`、`一句话入围理由`。
- 默认先按 `全球AI教育动态`、`AI教育技术应用资讯`、`AI技术前沿及资讯`、`国内“AI+教育”探索` 四个板块展示候选池，再视需要附一版 `建议优先写入 / 可替换备选` 汇总表。
- 若候选较多，可分成 `建议优先写入` 与 `可替换备选` 两组；但都要给出可选择序号。
- 候选清单后默认附一段 `覆盖说明`，简要交代本轮是否已补到 `教育部`、`欧洲`、`日本`、`国际组织`、`国外权威媒体涉华`；若其中某条线无合格命中，必须明确写原因。
- 候选清单展示后，明确提示用户“请从中选择要写入正式周报的5条资讯，可直接回复序号或标题”。
- 如果用户回复“就按你推荐的前五条写”，才把当前推荐前五自动转入正式写作名单。
- 在未获得用户确认前，禁止擅自把推荐前五当成正式成稿名单。

### 5. 用户确认正式写作名单
- 用户选定后，先复述并锁定这 `5` 条资讯，确认没有重名、重复事件或超出 `5` 条。
- 若用户选了少于 `5` 条，提醒其补足；若多于 `5` 条，提醒其缩减或明确允许扩写。
- 只有在名单锁定后，才进入正式成稿阶段。

### 6. 生成深度分析段
- 在撰写 `思考与启发`、`对教育的影响`、`对人才培养与就业的意义` 前，先读取 sibling skill [education-insight-architect](../education-insight-architect/SKILL.md)。
- 先为每条入选资讯整理事实包，至少包含：`标题`、`1-2句事实摘要`、`3-6条已核验事实`、`类别`、`信源等级`、`总分`、`进入正式5条的原因`。
- 在正式写 `思考与启发` 前，先额外提炼这条资讯最值得关注的 `3-4` 个主轴；若用户已明确给出关注点，优先直接吸收进首句主判断，不要改写成更泛的抽象表述。
- 若首句主判断超过 `55字`，优先先删减，再进入正式写作。
- 若条目来自国外，正式写作前还要补问一句：它对中国基础教育推进路径的启发是什么，哪些做法可借鉴、哪些不能照搬。
- 若条目涉及企业参与教育变革，还要补问一句：企业在这条资讯中承担的是能力支持、平台供给、教师培训还是实施服务角色，其价值与边界分别是什么。
- 由于本 skill 正式输出固定为 `5` 条，若需要统一写这五条的分析段，优先启用只读 subagent 批量生成分析字段。
- subagent 只产出分析字段，不得补新事实、改标题、扩写背景；主 skill 回收结果后再统一做长度、语气和禁用句式检查。

### 6.1 分析段字面质检
- 在任何 `思考与启发` 进入正式成稿前，必须额外做一轮 `literal-level lint`，不要只凭语感判断“应该差不多了”。
- 长度检查：普通条目严格控制在 `150-200字`，核心 `1-2` 条严格控制在 `250-350字`；若低于对应下限，通常仍像摘要体，需继续补足机制、条件或边界，不要把“写短”误当“写深”。
- 分段检查：核心 `1-2` 条若拆段，首段不能只有一句主判断；必须让读者在第 `1` 段就同时读到主判断与机制解释，第 `2` 段再承接实施重点、边界或跟踪点。
- 禁用句式搜索：逐项检查并删除 `不是……，而是……`、`不只是……，而是……`、`不仅……，而是……`、`重点不在……，而在……`、`关键不在……，而在……`、`这意味着`、`X的信号是`、`公司公告显示` 等句式及其变体。
- 教育术语精度检查：若内容本质在谈治理层级、区域推进、课程部署、扩散机制或行政协同，优先使用 `区域教育治理`、`区域课程工程`、`教育行政体系`、`市—区—校机制` 等词；只有当地理尺度本身就是判断核心时，才保留 `城市级` 等表述。
- 政策语气检查：若条目属于教育部、省级教育主管部门或正式部署性政策，分析段应聚焦落实重点、组织抓手和协同要求，不写成对政策本身的批判、审判或“成败由你证明”的上位评语。
- AI角色检查：若材料已把 AI 放入全系统部署，需明确判断其是 `公共能力 / 基础能力 / 制度能力` 还是单点工具，避免只写成一般数字化升级。
- 业务流检查：若条目涉及平台升级、能力中心或服务中心，需补看其是否进入 `备课、学习支持、教师发展、治理服务` 等真实业务流，避免停留在“平台上线”层。

### 7. 生成正式成稿
- 正式成稿标题统一使用 `全球AI教育动态周报`。
- 正式成稿不再按来源类别分章节，而是统一输出 `本期最值得关注的5条资讯`。
- 总览表默认使用 `序号`，不使用 `排名`。
- 总览表默认列为 `序号`、`资讯`、`类别`、`核心启示`；其中 `核心启示` 必须控制在 `20字以内`。
- 评分默认只保留在后台，不在总览表、正文或标签行中显示；只有用户明确要求看评分细节时才展示。
- 若用户明确要求“只给我 5 条，不要表格”，可以省略总览表，但仍按用户最终确认的五条输出。
- 若本期存在明确可视化价值的数据结构，如 `时间窗内事件密度`、`国家/地区分布`、`政策/课程/治理类别分布`、`推进层级分布`、`发布日期节奏`、`主体类型分布` 等，且图表确实比文字更利于理解，必须主动读取 sibling skill [visualization](../../visualization/SKILL.md) 并生成真实图表文件，再用于 HTML 页面或相关交付。
- 只有当图表没有信息增量、会重复读者一眼可见的结构信息、或数据规模过小不值得成图时，才允许只保留表格或时间线。
- 除非用户明确要求做结构统计图，否则优先把真正来自资讯正文的关键数字，如覆盖学校数、教师数、学生数、预算、课时、参与机构数、比例变化、调查结果或平台用户规模，转化为图表；若没有这类数字，就不要为了凑版面硬作图。
- 若图表生成后出现 `小字难读`、`比例标签不清`、`低值被压扁`、`不同量纲误放同轴` 等问题，必须继续优化，直到读者能快速看清关键数字；不能因为“已经有图”就停止。

### 8. 默认落盘
- 除非用户明确要求“只在对话里展示、不落盘”，否则正式交付默认在用户选定五条后，于 `docs/周报/` 下为本次生成创建独立文件夹。
- 文件夹名优先使用 `全球AI教育动态周报_YYYYMMDD_YYYYMMDD`。
- 若 `docs/周报/` 中已存在同名文件夹，不要覆盖原文件夹；必须按同目录递增后缀创建 `全球AI教育动态周报_YYYYMMDD_YYYYMMDD_2`、`..._3` 等。
- Markdown 默认保存为 `docs/周报/[批次文件夹名]/[批次文件夹名].md`。
- 生成 Word 时，必须保存为 `docs/周报/[批次文件夹名]/[批次文件夹名].docx`，与 Markdown 保持一一对应。

### 9. 要主动输出 Word
- 在保存 Markdown 的同时，导出同一批次文件夹内的同名 `.docx` 文件。
- Markdown 首行必须写 `# 全球AI教育动态周报`，第二个非空行必须单独写日期范围。
- Word 默认样式：第一行标题固定为 `全球AI教育动态周报`，使用 `黑体`、`加粗`、`二号`、`居中`；第二行单独写日期范围，使用月日格式，不写年份，`黑体`、`二号`、`居中`。
- 一级标题使用 `黑体`、`加粗`、`三号`；正文默认使用 `仿宋`、`三号`。
- Word 正文与次级标题默认两端对齐、首行缩进 `2字符`，全文固定 `28磅` 行距。
- 表格表头和表格正文使用 `仿宋`、`四号`，表格内部使用 `1.0` 单倍行距。
- `附：参考来源` 可保留在 Markdown 中，但不强制写入 Word。

### 10. 默认输出 HTML 页面
- 在生成 Markdown 和 Word 后，默认必须输出 `HTML` 页面或可视化长图文版。读取 sibling skill [document-to-html-presentation](../../document-to-html-presentation/SKILL.md) 来执行此操作。
- HTML 页面默认保存到同一批次文件夹，文件名使用 `docs/周报/[批次文件夹名]/[批次文件夹名]_可视化长图文报告.html`。
- HTML 页面默认沿用 `本期判断`、`资讯快览`、`逐条拆解` 三段式重组，不再机械照搬 Markdown 小标题层级。
- `附：参考来源` 可以保留在 Markdown 中，但默认不要渲染进最终 HTML 页面；只有用户明确要求在页面展示来源时才保留。
- HTML 页面中的正文段落、卡片文字、逐条拆解说明默认使用 `两端对齐`，不要出现左参差的正文排版。
- 若 HTML 页面附带导出长图功能，导出态也要保持正文 `两端对齐`，不要因为切换导出样式而丢失。
- 若 HTML 页面中适合加入图表，必须优先使用 `visualization` skill 生成图表资产，再由 HTML 页面引用；不要把“图表模块”退化成纯色块、假图占位或只是把表格改成卡片。

## 风格要求
- 标题克制、判断明确、语言专业。
- 用 `值得关注`、`提示了`、`显示出`、`当前公开信息不足以支持更强判断` 等表达控制语气。
- 不编造日期、数字、引语和政策口径。
- 不把企业发言直接写成行业共识。
- 不把国际案例直接等同于中国可复制路径，要明确前提差异。
- `思考与启发` 要体现制度设计、角色变化、业务流重构或实施前提，不能只是事实复述或评论员式感想。
- HTML 页面正文默认两端对齐，且不显示 `附：参考来源` 区块。

- 候选池中每条资讯都已记录总分或至少五维评分

- 候选清单已给出足够信息让用户选择：`候选序号`、`标题`、`日期`、`机构/地区`、`类别`、`总分或推荐等级`、`一句话理由`
