# 全球AI教育动态周报示例

## 示例1：标准版，两阶段输出
用户请求：
`生成过去7天的全球AI教育动态周报，先筛选候选资讯给我，我来决定写哪五条。`

建议处理：
- 先广泛建池，再统一评分，不要边搜边写定稿
- 候选池默认争取达到 `12-20` 条，再按 `100分制` 排序
- 虽然最终不分模块，但建池时仍要覆盖 `国际政策治理`、`AI技术与报告`、`AI教育技术应用`、`国内地方探索`
- 第一轮回复只输出 `候选清单`，至少包含 `候选序号`、`资讯`、`日期`、`地区/机构`、`类别`、`总分`、`一句话理由`
- 明确请用户从候选中选择要写入正式周报的 `5` 条
- 在用户确认前，不写 `本期判断`、不展开 `事件/思考与启发`、不导出 Markdown/Word/HTML
- 用户确认后，再生成正式周报并落盘

## 示例2：教育官员导向版
用户请求：
`给我做一版给教育局领导看的全球AI教育动态周报，先把候选新闻筛出来，我来定最后5条。`

建议处理：
- 评分时提高 `系统重要性` 与 `证据强度` 的权重敏感度
- 若总分接近，优先保留国家级、区域级、制度级动作
- 候选清单中的 `一句话理由` 优先概括治理价值、政策抓手和规模化意义
- 用户确认名单后，`思考与启发` 优先写政策设计、区域推进节奏、师训机制和规模化条件

## 示例3：校长导向版
用户请求：
`给我一版适合校长看的AI教育周报，先给候选清单，我来选5条，每条都要有学校实施启发。`

建议处理：
- 候选评分时重点看课程实施、教师工作流、校园治理、家校沟通和接入成本
- 候选清单中优先保留真正进入学校业务流的条目
- 用户确认名单后，`思考与启发` 优先解释哪些环节已经进入真实业务流，哪些仍停留在宣传层

## 示例4：只在对话中展示
用户请求：
`整理一份本周全球AI教育动态周报，先给我候选新闻，我选5条，最后直接在对话里给我，不要落盘。`

建议处理：
- 保留完整建池和评分流程
- 第一轮回复先停在候选清单，不直接生成正式稿
- 正式输出不显示 `评分`
- 省略落盘动作，但不要省略 Markdown 中的参考来源
