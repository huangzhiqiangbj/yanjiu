# 调用示例

## 示例 1：政策速答
用户：
```text
目前中国对生成式AI进入中小学课堂，有没有明确的政策边界？
```

处理方式：
- 先判断这是 `政策速答 + 边界判断`
- 先核中央和教育主管部门官方文件
- 若涉及未成年人、算法、生成服务，再补看网信等相关部门规则
- 输出 `结论 -> 政策依据 -> 对学校和教师的含义 -> 仍需核实`

## 示例 2：政策时间线
用户：
```text
帮我梳理一下中国“教育数字化”近几年的政策时间线，给局领导看。
```

处理方式：
- 先判断这是 `时间线梳理 + 领导简报`
- 优先找顶层部署，再找教育部和地方推进文件
- 不要只罗列文件名，要写出 `关键转折` 和 `当前主口径`
- 默认输出 `核心判断 + 时间线 + 影响分析 + 观察点`

## 示例 3：中央与地方比较
用户：
```text
比较一下教育部关于教育App入校的要求，和某省最近的地方实施细则有什么差别。
```

处理方式：
- 先判断这是 `政策比较`
- 先用上位规则定位中央基线，再看地方细则如何细化
- 明确哪些差异只是执行层面，哪些差异会改变学校和企业动作
- 输出 `共同点 -> 差异点 -> 对学校/企业的意义 -> 仍需补核`

## 示例 4：企业边界快检
用户：
```text
我们做一个给中小学生的AI答疑产品，这在“双减”和教育监管下有什么边界？
```

处理方式：
- 先判断这是 `边界 / 合规快检`
- 先分清产品处于 `校内` 还是 `校外`，是 `工具`、`课程` 还是 `培训服务`
- 核政策时不要只看教育部，还要看校外培训治理、未成年人保护、数据治理等相关规则
- 输出 `初步结论 -> 明确红线 -> 可做但有条件 -> 仍需核实`

## 示例 5：政策简报
用户：
```text
写一份“中国AI+教育政策动态”的简短政策简报，面向教育科技公司管理层。
```

处理方式：
- 先判断这是 `政策简报`
- 只基于官方文件和已核信息写
- 重点写 `政策重心变化`、`对产品方向的影响`、`需要规避的边界`
- 不写成空泛行业评论

## 示例 6：地方执行判断
用户：
```text
中央政策已经鼓励人工智能进校园了，是不是学校现在都可以自由上各种AI工具？
```

处理方式：
- 先识别这类问题常见误区：把 `政策鼓励` 误当成 `学校无限制使用`
- 回答时必须拆开：
  - 中央总体方向
  - 地方与学校准入规则
  - 数据、内容、未成年人、采购与管理责任
- 给出结论时要明确 `鼓励方向` 不等于 `无边界放开`

## 示例 7：二十大以来主线解读
用户：
```text
你帮我讲清楚，二十大以来中国教育政策的主线到底是什么？
```

处理方式：
- 先判断这是 `政策母体解读`
- 不要直接罗列通知，而要先回到 `二十大报告 -> 二十届三中全会 -> 教育强国纲要 -> 政府工作报告`
- 先讲 `教育、科技、人才一体统筹`，再讲 `教育强国`、`高质量教育体系`、`教育数字化`、`AI+教育`
- 最后再落到对基础教育、区域推进、学校治理和企业边界的含义

## 示例 8：近五年政府工作报告比较
用户：
```text
把近五年政府工作报告里关于教育的表述变化梳理出来，我想看政策重心是怎么迁移的。
```

处理方式：
- 先判断这是 `政策话语解码 + 时间线梳理`
- 不要只摘出每年的教育段落，要比较 `新增表述`、`排序变化`、`政策重心迁移`
- 默认从 `公平与治理`、`高质量体系`、`教育强国`、`数字教育`、`AI+教育` 这几条线去看
- 输出 `总判断 -> 年度演进 -> 核心变化 -> 启示`

## 示例 9：教育强国与AI+教育关系
用户：
```text
从《教育强国建设规划纲要》和教育数字化政策看，中国为什么要推AI+教育？
```

处理方式：
- 先判断这是 `政策母体解读 + 影响分析`
- 先回答 AI+教育 在 `教育强国、科技强国、人才强国` 中的位置
- 再解释它如何进入 `课程、教材、教学、评价、教师发展、资源平台、治理安全`
- 不要把 AI+教育 讲成单一产品机会，要同时讲公共平台、教师能力和治理边界

## 示例 10：十五五规划中的教育判断
用户：
```text
十五五规划里，教育有哪些新增抓手？和十四五后期、教育强国纲要是什么关系？
```

处理方式：
- 先判断这是 `政策母体解读 + 政策比较`
- 先核正式 `十五五` 纲要文本，再把它放回 `教育强国纲要` 和近年政府工作报告里一起看
- 重点看 `教育科技人才一体发展`、`基础教育扩优提质`、`学科专业布局`、`教师队伍`、`前沿领域人才培养`
- 输出时明确哪些是延续项，哪些是新增抓手，哪些是节奏明显前移的部署

## 示例 11：先查政策库再调原文
用户：
```text
把教育强国纲要、近五年政府工作报告和教育数字化政策里关于AI+教育的原文都调出来，做个比较。
```

处理方式：
- 先判断这是 `政策原文调用 + 政策话语解码 + 政策比较`
- 第一步不是直接搜网页，而是先读 `policy-corpus.md`
- 先从库里定位 `T03`、`G22-G26`、`D02`、`A01`、`A02`
- 再逐个打开官方链接，抽取关键原文表述
- 输出时区分 `原文`、`解释`、`边界`，不要把概括写成原文

## 示例 12：各省细则比较
用户：
```text
把各省教育数字化和AI+教育细则梳理一下，看看哪些省已经有专项方案。
```

处理方式：
- 先判断这是 `省际比较 + 地方细则梳理`
- 第一步先读 `province-policy-corpus.md`
- 先区分 `专项方案完整型`、`总方案牵引型`、`平台/试点推进型`、`待补原始页型`
- 对重点省份再回源对应官方入口
- 输出时不要假装所有省份政策成熟度一致，差异本身就是结论

## 示例 13：北京深挖
用户：
```text
别泛泛而谈，把北京AI+教育政策挖深一点，讲清楚到底是怎么落地的。
```

处理方式：
- 先判断这是 `重点省份深挖`
- 第一步先读 `beijing-policy-deep-dive.md`
- 不要只列北京有哪几个文件，要讲清 `应用工作方案 -> 中小学专项方案 -> 地方课程纲要 -> 专家委员会` 之间的关系
- 回答时重点写 `课时、课程模块、师资计划、平台超市、产品评测、综合素质评价`

## 示例 14：关键省份原始文件比较
用户：
```text
只基于教育厅或省政府原始文件，比较北京、上海、江苏、广东、山东、四川的AI+教育政策。
```

处理方式：
- 先判断这是 `关键省份原始文件比较`
- 第一步先读 `key-provinces-ai-education-source-trace.md`
- 再逐个打开原始文件页，不用媒体稿和转发稿替代
- 输出时区分 `原始文件明确写了什么`、`你从中读出的政策差异`、`哪些省还需要继续补核`
