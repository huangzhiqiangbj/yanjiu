# K-12 AI Research Radar Reference

## 一、来源架构
做基础教育 AI 研究时，先区分 `检索工具`、`学术源`、`机构源`、`发现型来源` 四类对象。

### 1. 检索工具
这些工具只负责发现候选，不直接代表质量：
- `Google Scholar`
- `Web of Science`
- `Scopus`
- `ERIC`
- `Crossref`
- `CNKI（知网）`
- `万方`

中文期刊默认优先级：
1. `CNKI（知网）`
2. `官网目录页`
3. `万方`
4. `维普`

说明：
- 对 `教育研究`、`中国电化教育`、`电化教育研究` 这类中文核心，默认先用 `CNKI/知网`
- 不要只依赖通用搜索引擎抓中文核心目录页和文章元数据
- 若 `CNKI/知网` 暂未暴露近月条目，再补官网目录页和其他数据库

### 2. 国际高优先级学术源
优先用于建立核心判断。

| 类别 | 推荐来源 | 适合用途 |
| --- | --- | --- |
| 教育技术综合 | `Computers & Education` | 技术应用、学习效果、系统变革 |

| 教育技术综合 | `Educational Technology Research and Development` | 教育技术研究与实施机制 |

| AI教育专门 | `Computers and Education: Artificial Intelligence` | 生成式AI与教育前沿 |

| 会议 | 适合用途 |
| --- | --- |
| `AIED` | AI in education 核心学术前沿 |
| `LAK` | 学习分析、数据驱动教育改进 |
| `EDM` | 教育数据挖掘、学习过程与预测 |
| `Learning at Scale` | 规模化学习平台与实施问题 |

补充原则：
- 会议论文适合发现新议题、早期机制和方法趋势
- 若要形成稳定判断，优先继续追踪其后续期刊化成果

这些材料适合补政策语境和国际框架，不替代同行评审论文。

| 机构 | 适合用途 |
| --- | --- |
| `UNESCO` | AI与教育国际原则、治理框架、全球进展 |
| `OECD` | 教育政策、技能、系统变革、国际比较 |
| `World Bank` | 教育系统改革、发展中国家实施路径 |
| `UNICEF` | 儿童权益、数字安全、未成年人保护 |
| `RAND` | 政策研究、学校实施、制度分析 |
| `European Commission/JRC` | 欧盟教育数字化与治理框架 |

### 6. 发现型来源
可用于发现议题，但默认不能直接当作正式证据：
- 企业白皮书
- 公司研究博客
- 咨询报告
- 媒体专访
- 行业大会演讲

使用规则：
- 只用来发现问题、术语、产品方向和潜在案例
- 若观点重要，必须回到学术源或权威机构源做二次核对

## 二、默认选题分桶
面对“AI在基础教育中的应用、趋势、现状、变革”这类大话题时，默认先拆成以下研究桶，而不是一次性混写。

1. `技术变革`
   - 生成式AI、多模态、智能体、学习分析等到底改变了什么教育变量
2. `政策影响`
   - 它如何影响教育治理、资源配置、评价安排、合规边界
3. `实践路径`
   - 学校、区域、平台如何从试点走向常态化
4. `风险与治理`
   - 未成年人保护、数据、内容安全、责任归属

若用户没有特别指定，优先用前 `3` 个桶构建主线，把 `风险与治理` 作为约束条件写入。

## 四、检索与筛选顺序
1. 先找 `综述/研究回顾` 建立全景，不要一上来就陷入单篇案例。
2. 再找 `代表性实证研究` 判断证据强弱。
3. 再找 `政策/治理论文或机构报告` 判断制度边界。
4. 最后再看 `学校或区域实践案例`，判断实施条件和复制边界。

初筛时优先问：
- 这篇是否真的是 `基础教育/K-12`
- 这篇是否真正在讨论 `AI`，而不是一般数字化
- 它更偏 `技术变革`、`政策影响` 还是 `实践路径`
- 它是证据文献、概念论文，还是评论性文章
- 它的结论是基于什么材料得出的

## 五、评分 rubric
每篇候选文献按 `100` 分评分，五项各 `20` 分。

### 1. 来源权威性
- `18-20`：该领域高质量同行评审期刊、核心会议，或方法透明的一线公共机构报告
- `12-17`：主流期刊、较可靠会议、正规研究机构报告
- `6-11`：来源一般，或虽正规但影响力和透明度有限
- `0-5`：来源不清、非同行评审、营销或宣传性质明显

### 2. 主题贴合度
- `18-20`：直接研究基础教育 AI，且正面回应技术变革、政策影响或实践路径
- `12-17`：与基础教育 AI 高度相关，但学段或问题边界略混杂
- `6-11`：与教育 AI 有关，但更多是高教、成人教育或宽泛数字化
- `0-5`：只在标题或引言中提到 AI/教育，实质不贴题

### 3. 证据强度
- `18-20`：系统综述、元分析、纵向、多案例、多区域、准实验或证据链完整
- `12-17`：调查、案例研究、设计研究或数据分析较完整
- `6-11`：证据单薄、样本较窄、主要是初步观察
- `0-5`：观点型、评论型或几乎无明确证据支撑

### 4. 结构洞察
- `18-20`：能解释机制、揭示边界、指出规模条件或治理逻辑
- `12-17`：有一定解释力，但仍偏描述
- `6-11`：主要停留在趋势描述、现象归纳或价值判断
- `0-5`：只有表态，没有结构分析

### 5. 中国迁移价值
- `18-20`：能明确帮助判断中国教育场景下的制度前提、可迁移部分和不可迁移部分
- `12-17`：有较强参考价值，但需要较多再解释
- `6-11`：参考意义有限，制度背景差异较大
- `0-5`：几乎无法迁移，或迁移风险高

补充规则：
- 对政策/治理论文，不要因为它不是实验研究就机械低分
- 对学校实践案例，不要因为样本小就一票否决；关键看它是否说清了条件和边界

## 六、证据类型的默认优先级
优先级只用于帮助排序，不是绝对结论。

1. `系统综述 / 元分析`
2. `高质量综述 / 领域回顾`
3. `纵向研究 / 多区域研究 / 准实验`
4. `多案例学校或学区研究`
5. `单案例研究 / 调查研究 / 设计研究`
6. `概念论文 / 观点论文 / 评论文章`

使用提醒：
- 研究综述适合建立“有哪些共识与分歧”
- 政策论文适合建立“制度如何组织”
- 实践案例适合判断“规模化需要什么条件”

## 七、什么样的观点值得留下
优先保留能完成以下动作的观点：
- 把问题从“某工具是否有效”上升到“教育系统的哪个变量被改写”
- 把问题从“教师用不用 AI”上升到“教师、学校、平台、政策之间如何重新分工”
- 说明为什么某类试点可以成立，但未必能规模化
- 识别政策与治理准备是否落后于技术扩散速度
- 区分全球共性问题与特定制度环境问题
- 说明中国场景下哪些可以借鉴，哪些不能照搬

## 八、什么样的观点应降权
- 只有宏大判断，没有证据来源
- 把企业案例直接写成行业趋势
- 把高教结论直接平移到基础教育
- 把政策表述直接当成现实落地结果
- 把一国案例直接当成普遍路径
- 只说“AI很重要”，但没有机制、边界和条件

## 十三、默认回应策略
### 1. 用户要“高质量学术源”
优先输出：
- 来源分层
- 建议长期跟踪的期刊、会议、机构
- 每类来源为什么重要

### 2. 用户要“高质量论文”
优先输出：
- 候选清单
- Top 5 或 Top 10 论文
- 每篇为什么入选
- 至少 `2-3` 篇做深读论文卡

### 3. 用户要“好观点”
优先输出：
- 观点卡
- 观点之间的共识与分歧
- 观点的证据强弱
- 哪些观点对中国教育更有解释力

### 4. 用户要“趋势/现状/变革”
优先输出：
- 核心判断
- 研究共识
- 新旧变化点
- 尚未解决的问题
- 政策和实践路径的含义

## 十四、常见错误
- 把数据库当成高质量来源本身
- 只用开放Web搜索中文核心，而不先查 `CNKI/知网`
- 只给题目列表，不说明为什么值得读
- 只复述摘要，不做研究判断
- 混用基础教育、高教、职业教育结论
- 把“技术应用”写成“教学设计优化”而偏离用户重点
- 忽略政策、治理和规模化条件
- 看到热点词就默认它代表长期趋势

## 十五、建议的最小交付包
如果用户没有指定格式，默认给出以下三层结果：
1. `来源建议`：先给可靠学术源池
2. `核心论文`：给高分候选与简要说明
3. `观点沉淀`：给最值得长期跟踪的判断及其边界
