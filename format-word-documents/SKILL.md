# Word 文档格式规范化

## 核心定位
专门用于修改已有或新导出的 Word 文档格式，优先处理报告、周报、简报等正式文档的版式统一问题。

## 何时使用
- 用户要求“修改 Word 格式”“统一 docx 排版”“调整标题、正文、表格样式”
- 用户要求把 Markdown 导出的 Word 修成正式版
- 用户要求清理项目符号、斜体、英文双引号或中文冒号后的空格

## 默认格式规则
- 文档标题：`黑体`、`加粗`、`二号`、`居中`
- 标题下日期：另起一行，只写`月日`，不写年份；`黑体`、`二号`、`居中`
- 一级标题（如`一、二、三、四、五`）：`黑体`、`加粗`、`三号`
- 次级标题：`仿宋`、`加粗`、`三号`，首行缩进 `2字符`
- 正文：`仿宋`、`三号`、固定 `28磅` 行距
- 正文段落：首行缩进 `2字符`
- 全文所有段落默认删除 `段前` 与 `段后` 间距，统一设为 `0磅`
- 标签行（如`事件：` `思考与启发：` `成效：` `内容提炼：`）：整段首行缩进 `2字符`，缩进放在标签前，不放在标签后
- 默认不要项目符号；若来源文本含 `-`、`•` 等列表标记，改为普通段落或编号段落
- 默认不用斜体；需要强调时优先使用加粗
- `事件`、`思考与启发`、`对教育的影响`、`成效`、`内容提炼`、`对人才培养与就业的意义` 等标签默认加粗

## 表格规则
- 表头和表格正文都使用 `仿宋`、`四号`
- 表格内部使用 `1.0` 单倍行距，不沿用正文 `28磅` 固定行距
- 表格内段落默认使用 `两端对齐`，不要保留居中对齐
- 表格内段落同样删除 `段前` 与 `段后` 间距，统一设为 `0磅`
- 若表格过宽，优先压缩措辞或拆表，不靠缩小字号塞内容
- 尽量避免表格跨页；必要时拆表，避免同一行被分页切断

## 文本清洗规则
- 将自然语言正文中的英文双引号 `"` 统一改为中文双引号 `“”`
- 只对自然语言正文做引号替换；代码、URL、文件路径、命令和技术标识符保持原样
- 去除中文冒号 `：` 后的空格，包括半角空格、全角空格和连续空白，以及其他不必要的空格
- 清理中文自然语言中的多余空格，尤其是汉字与英文缩写、英文单词、数字之间的无意义空格
- 标签名、标题名、表格单元格文本都要去掉首尾多余空格，例如 `政策伦理维度 ` 改为 `政策伦理维度`
- 标签行统一写成 `事件：正文`，不要写成 `事件： 正文`
- 引号内文本同样执行冒号后空格清理，例如 `“标题： 内容”` 改为 `“标题：内容”`

## 常见替换
- `"人工智能"` → `“人工智能”`
- `事件： 这是正文` → `事件：这是正文`
- `“标题： 内容”` → `“标题：内容”`
- `由 UNESCO 于 2026 年发布` → `由UNESCO于2026年发布`
- `政策伦理维度 ` → `政策伦理维度`

## 处理流程
1. 识别文档结构，区分标题、日期、一级标题、次级标题、正文、标签行和表格。
2. 先做文本清洗，再做样式调整；不要先套样式后再批量替换文本。
3. 清除误导出的项目符号、斜体和多余空格。
4. 把全文段前段后统一清零，再处理首行缩进、行距和对齐方式。
5. 单独处理表格字体、字号、行距和对齐方式，不要让表格继承正文样式。
6. 最后逐项检查缩进、加粗、引号、冒号空格、分页表现和表格对齐。

## 特别注意
- 若文档来自 Markdown 导出，重点检查列表标记是否被错误渲染成 Word 项目符号。
- 若标题层级不完整，不要为了形式保留孤立编号或空标题。
- 若某段是公式、代码、路径或英文技术字符串，不要强行改成中文标点。

## 质量检查
交付前确认：
- 标题、日期、一级标题、次级标题和正文样式都已统一
- 全文段前与段后间距都已清零
- 正文与标签行都已按首行缩进 `2字符` 处理
- 文档中没有残留项目符号和斜体
- 表格已改为 `仿宋`、`四号`、`1.0` 行距
- 表格单元格内段落已改为 `两端对齐`
- 英文双引号已替换为中文双引号
- 中文冒号后的空格已清理干净
- 汉字与英文、数字之间的多余空格已清理干净
- `事件：`、`思考与启发：` 等标签后不再保留空格
- 表格未因内容过长而明显撑宽或跨页失控
